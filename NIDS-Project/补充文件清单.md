## 一、已创建文件清单

### 1. 样本平衡实验脚本
**文件**: `backend/check_balance.py`

**功能**:
- 随机欠采样（Random Undersampling）
- 随机过采样（Random Oversampling）
- 三种策略对比（Baseline vs Undersampling vs Oversampling）
- 自动生成对比图表

**输出**:
- `outputs/balance_experiment_results.csv` - 实验结果表
- `outputs/experiment_comparison.png` - 指标对比图
- `outputs/class_distribution_comparison.png` - 类别分布对比
- `outputs/f1_score_comparison.png` - F1分数对比

**使用方法**:
```bash
cd backend
python check_balance.py
```

## 七、注意事项

### 1. 数据集路径
- 确保 NSL-KDD 数据集位于 `backend/dataset/` 目录
- 文件名必须是 `KDDTrain+.txt` 和 `KDDTest+.txt`

### 2. 模型文件
- 运行 `05_model_comparison.py` 前，确保已训练模型
- 模型文件应位于 `backend/models/` 目录

### 3. 运行时间
- `check_balance.py`: 约 5-10 分钟（取决于数据集大小）
- `05_model_comparison.py`: 约 2-5 分钟（取决于模型数量）
- `generate_charts.py`: 约 10-30 秒

### 4. 内存占用
- 样本平衡实验：约 2-4 GB
- 模型对比：约 1-2 GB
- 图表生成：约 500 MB

## 八、常见问题

### Q1: 找不到数据集文件
**A**: 检查数据集是否在以下路径之一：
- `dataset/KDDTrain+.txt`
- `backend/dataset/KDDTrain+.txt`
- `../dataset/KDDTrain+.txt`

### Q2: 找不到模型文件
**A**: 先运行训练脚本：
```bash
cd backend
python train_model.py
python train_hybrid.py
python train_cnn_lstm.py
```

### Q3: 图表中文显示乱码
**A**: 安装中文字体：
- Windows: 已内置 SimHei
- Linux: `sudo apt-get install fonts-wqy-zenhei`
- macOS: 使用 Arial Unicode MS

### Q4: TensorFlow 相关错误
**A**: 确保安装了正确版本的 TensorFlow：
```bash
pip install tensorflow>=2.10.0
```
