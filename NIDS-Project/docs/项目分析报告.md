## 三、关键问题诊断

### 3.1 技术架构问题
**问题1：缺少深度学习框架**
- 当前：仅使用scikit-learn
- 需要：TensorFlow/PyTorch
- 影响：无法实现CNN-LSTM、Transformer等深度模型

**问题2：特征工程简化**
- 当前：基础的NSL-KDD特征（41维）
- 需要：BERT嵌入、TF-IDF、时序特征
- 影响：特征表达能力不足

**问题3：缺少强化学习模块**
- 当前：静态检测
- 需要：TD3动态策略优化
- 影响：无法实现自适应防御

### 3.2 数据处理问题
**问题4：数据不平衡未充分解决**
- 当前：基础SMOTE（代码中未见实现）
- 需要：自适应SMOTE + 边界样本合成
- 影响：召回率偏低（62.87%）

**问题5：缺少对抗样本训练**
- 当前：无对抗训练
- 需要：PGD、FGSM对抗样本生成
- 影响：模型鲁棒性不足

from imblearn.over_sampling import SMOTE

def balance_dataset(X, y):
    smote = SMOTE(sampling_strategy='auto', random_state=42)

import time

class PerformanceMonitor:
    def measure_latency(self, func):
        start = time.time()
        result = func()
        latency = (time.time() - start) * 1000  # ms
        return result, latency

## 八、风险与挑战

### 8.1 技术风险
1. **深度学习模型训练困难**
   - 风险：过拟合、训练不收敛
   - 缓解：使用预训练模型、数据增强

2. **强化学习环境设计复杂**
   - 风险：奖励函数设计不当
   - 缓解：参考现有论文、逐步调试

3. **实时性能不达标**
   - 风险：推理延迟过高
   - 缓解：模型量化、批处理优化

### 8.2 数据风险
1. **数据集过时**
   - NSL-KDD是2009年的数据集
   - 建议：添加CIC-IDS2017等现代数据集

2. **数据不平衡严重**
   - 当前召回率仅62.87%
   - 建议：SMOTE + 焦点损失函数
