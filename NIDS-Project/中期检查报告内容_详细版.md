# 一、项目建设目标

## 1.1 总体目标

本项目旨在研究并开发一套基于人工智能技术的轻量级网络入侵检测系统（Network Intrusion Detection System, NIDS），通过融合传统机器学习与深度学习方法，实现对网络流量中异常行为和攻击活动的自动检测与预警。

## 1.2 具体目标

### （1）数据处理与特征工程目标
- 完成NSL-KDD等公开数据集的获取、清洗和预处理工作
- 实现42维网络流量特征的标准化处理和类别编码
- 探索多模态特征融合方法，提升特征表达能力

### （2）检测模型构建目标
- 训练基于Random Forest的基线检测模型，达到75%以上的检测准确率
- 探索CNN-LSTM混合深度学习模型，尝试提升检测性能
- 实现规则检测与AI检测相结合的混合检测模式

### （3）系统开发目标
- 开发基于Flask的后端API服务，提供检测控制、日志管理、统计分析等功能
- 开发Web端可视化Dashboard，实时展示检测结果和威胁态势
- 完成前后端系统集成，形成可演示的完整原型

### （4）学习与能力提升目标
- 掌握Python数据处理和机器学习基础
- 了解网络安全和入侵检测的基本原理
- 积累软件开发和项目管理经验

### 2.2.1 数据准备与预处理模块（已完成）

**（1）数据集获取**

成功获取并整合了NSL-KDD网络入侵检测数据集，该数据集是网络安全领域广泛使用的基准数据集。

| 数据集 | 样本数量 | 用途 |

**（2）数据预处理模块开发**

开发了完整的数据预处理模块（`data_preprocessor.py`，138行代码），实现以下功能：

- **特征定义**：定义了NSL-KDD数据集的42个特征列，包括：
  - 基础连接特征（9个）：duration, protocol_type, service, flag, src_bytes, dst_bytes等
  - 内容特征（13个）：hot, num_failed_logins, logged_in, num_compromised等
  - 流量统计特征（10个）：count, srv_count, serror_rate, same_srv_rate等
  - 主机统计特征（10个）：dst_host_count, dst_host_srv_count等

- **类别特征编码**：使用LabelEncoder对3个类别特征（protocol_type, service, flag）进行数值编码

- **数值特征标准化**：使用StandardScaler对39个数值特征进行Z-score标准化处理

- **标签二值化**：将多类攻击标签转换为二分类标签（0=正常，1=攻击）

**（3）特征工程流水线**

开发了特征工程流水线模块（`src/modules/features/pipeline.py`，69行代码），包含：
- 特征编码（Encoding）
- 特征融合（Fusion）
- 特征标准化（Scaling）
- 特征重要性分析（Analysis）
- 特征选择（Selection）

### 2.2.2 机器学习模型训练模块（已完成）

**（1）Random Forest基线模型**

开发了模型训练模块（`train_model.py`，132行代码），使用scikit-learn框架训练Random Forest分类器。

模型配置参数：
```
- 决策树数量（n_estimators）：100
- 最大深度（max_depth）：20
- 随机种子（random_state）：42
- 并行计算（n_jobs）：-1（使用全部CPU核心）
```

**模型性能指标（在测试集上）：**

| 指标 | 数值 | 说明 |
|-----|------|-----|
| 准确率（Accuracy） | 77.65% | 整体分类正确率 |
| 精确率（Precision） | 96.73% | 预测为攻击的样本中真正是攻击的比例 |
| 召回率（Recall） | 62.87% | 实际攻击样本中被正确识别的比例 |
| F1分数（F1-Score） | 76.21% | 精确率和召回率的调和平均 |

**混淆矩阵：**
```
              预测正常    预测攻击
实际正常       9,438        273
实际攻击       4,765      8,068
```

分析：模型的精确率较高（96.73%），说明当模型判断为攻击时，准确性很高，误报率低；但召回率相对较低（62.87%），说明有约37%的攻击样本未能被检测出来，存在漏报情况。

**（2）CNN-LSTM混合模型（探索中）**

开发了混合模型架构（`models/hybrid_model.py`，35行代码），采用CNN-LSTM结构：

```
模型架构：
Input → Reshape → Conv1D(64) → MaxPool → Conv1D(128) → MaxPool 
      → LSTM(64) → LSTM(64) → Dense(64) → Dropout(0.5) → Dense(1, sigmoid)
```

- CNN层：提取局部空间特征
- LSTM层：捕获时序依赖关系
- 全连接层：进行最终分类

模型已完成架构设计和训练脚本编写（`train_hybrid.py`，91行代码），正在进行训练调优。

# 四、目前存在问题及原因

## 4.1 模型性能方面

### 问题1：召回率偏低（62.87%）

**问题描述**：当前Random Forest模型的召回率为62.87%，意味着约37%的攻击样本未能被正确识别，存在较高的漏报率。

**原因分析**：
1. **数据不平衡**：NSL-KDD数据集中正常流量和攻击流量的比例不均衡，模型倾向于预测多数类
2. **调参经验不足**：团队对机器学习超参数调优经验有限，未能找到最优参数组合
3. **未使用数据增强**：尚未实现SMOTE等过采样技术来平衡数据集

### 问题2：深度学习模型进展较慢

**问题描述**：申报书中提到的CNN-LSTM混合模型、Transformer、强化学习（TD3）等高级技术尚未完全实现和验证。

**原因分析**：
1. **基础薄弱**：团队成员深度学习基础相对薄弱，学习曲线较陡
2. **计算资源限制**：深度学习模型训练需要较长时间和GPU资源
3. **优先级调整**：前期将主要精力投入到系统原型搭建，模型优化工作相对滞后

## 4.2 功能实现方面

### 问题3：实时流量采集功能未完成

**问题描述**：目前系统主要基于系统状态特征进行检测，尚未实现真实网络流量的实时采集和分析。

**原因分析**：
1. **技术难度大**：网络流量采集涉及底层网络编程，需要使用Scapy等专业工具
2. **环境配置复杂**：Scapy在Windows环境下需要安装WinPcap/Npcap，配置较为繁琐
3. **权限限制**：校园网环境下进行流量采集可能存在权限和合规性问题

### 问题4：部分高级功能未实现

**问题描述**：申报书中提到的一些高级功能尚未实现，包括：
- DPDK高性能数据包处理
- TLS 1.3加密通信
- STIX 2.1标准的威胁情报格式
- 红蓝对抗沙箱

**原因分析**：
1. **技术门槛高**：这些技术需要较深的专业知识和工程经验
2. **时间有限**：学期中课程任务较重，投入项目的时间有限
3. **目标调整**：根据实际情况，优先保证核心功能的实现

## 4.3 项目管理方面

### 问题5：进度略有滞后

**问题描述**：部分工作进度落后于原计划，特别是深度学习模型的训练和优化。

**原因分析**：
1. **时间估计不足**：对部分技术难点的工作量估计不足
2. **学业压力**：学期中课程考试和作业占用了较多时间
3. **团队协作**：远程协作效率有待提高
