# NeuroSec-Aegis 前端功能说明

## 🎯 已实现的功能

### 1. 自动检测与数据更新
- ✅ **自动刷新**: 每 5 秒自动从后端获取最新数据
- ✅ **实时统计**: 显示累积解析包数、威胁数量、威胁占比
- ✅ **系统状态**: 实时显示检测引擎运行状态（运行中/已停止）
- ✅ **时间显示**: 顶部导航栏显示当前系统时间

### 2. 控制按钮功能
#### 启动按钮 (重置并启动全域引擎)
- 功能: 启动后端检测引擎
- API: `POST /api/detection/start`
- 效果:
  - 系统状态变为"运行中"
  - 状态指示灯变为绿色
  - 开始实时检测网络流量

#### 停止按钮 (物理隔离/挂起引擎)
- 功能: 停止后端检测引擎
- API: `POST /api/detection/stop`
- 效果:
  - 系统状态变为"已停止"
  - 状态指示灯变为黄色
  - 停止检测但保留历史数据

#### 清空日志按钮 (净化终端日志)
- 功能: 清空前端显示的日志
- 效果: 清空日志显示区域，添加清空提示

### 3. 数据可视化

#### 核心指标卡片
1. **累积解析包数**: 显示总扫描包数和吞吐率
2. **捕获威胁实体**: 显示威胁总数和威胁占比
3. **AI 引擎状态**: 显示引擎运行状态和推理延迟
4. **最后阻断指令**: 显示最后扫描时间

#### 威胁态势图谱
- **进度条显示**:
  - DDoS洪泛攻击 (红色)
  - APT 隐蔽隧道异常 (黄色)
  - WebShell / SQL注入 (青色)
- **网络拓扑图**:
  - 显示 Aegis Core 和 4 个监控节点
  - 受攻击节点显示为红色
  - 每 4 秒自动更新节点状态

#### 实时日志流
- **日志类型**:
  - 🔵 态势 (info): 系统信息
  - 🟡 预警 (warn): 警告信息
  - 🔴 拦截 (error): 威胁拦截
- **自动滚动**: 新日志自动滚动到底部
- **日志限制**: 最多保留 100 条日志

### 4. 状态指示器
- **DEFCON 状态**:
  - DEFCON 3 (运行中) - 绿色脉冲
  - DEFCON 5 (已停止) - 黄色
  - DEFCON 1 (连接失败) - 红色

### 5. 通知系统
- **浮动通知**: 右上角显示操作结果
- **自动消失**: 3 秒后自动消失
- **颜色区分**:
  - 成功: 青色
  - 警告: 黄色
  - 错误: 红色

## 🔌 API 接口

### 后端地址
```
http://localhost:5000/api
```

### 使用的端点
1. `GET /api/stats` - 获取统计数据
2. `GET /api/logs?limit=50` - 获取日志
3. `GET /api/detection/status` - 获取检测状态
4. `POST /api/detection/start` - 启动检测
5. `POST /api/detection/stop` - 停止检测

## 🎨 界面特色

### 赛博朋克风格
- 深空背景 + 网格效果
- 玻璃态面板 (Glassmorphism)
- 扫描线动画效果
- 霓虹色彩 (青色、紫色、红色)

### 响应式设计
- 支持桌面和平板显示
- 小于 1024px 自动切换为单列布局

## 📝 使用流程

1. **登录系统**
   - 输入用户名和密码（当前为模拟登录）
   - 点击"建立安全连接"

2. **查看实时数据**
   - 系统自动每 5 秒刷新数据
   - 查看指标卡片、图表和日志

3. **控制检测引擎**
   - 点击"启动"按钮开始检测
   - 点击"停止"按钮暂停检测
   - 点击"清空日志"清理显示

4. **监控威胁**
   - 观察威胁态势图谱的进度条
   - 查看网络拓扑图的攻击节点
   - 阅读实时日志流的威胁信息

## 🔧 技术栈

- **前端框架**: 原生 JavaScript (ES6+)
- **图表库**: ECharts 5.4.3
- **图标库**: Font Awesome 6.4.0
- **字体**: Orbitron + Exo 2
- **构建工具**: Vite (开发服务器)

## 🚀 启动方式

```bash
# Windows
start_neurosec.bat

# Linux/macOS
./start_neurosec.sh
```

访问地址: http://localhost:5175

## 📊 数据流程

```
后端 (Flask) → API 端点 → 前端 JavaScript → 数据处理 → UI 更新
     ↓                                              ↓
  检测引擎                                      图表/日志
```

## ⚡ 性能优化

- 使用 `Promise.all()` 并行请求多个 API
- 日志限制为 100 条，防止内存溢出
- 图表自动响应窗口大小变化
- 定时器在页面卸载时自动清理
